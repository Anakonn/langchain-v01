# AWS

[Amazon AWS](https://aws.amazon.com/)プラットフォームに関連する`LangChain`インテグレーション。

`langchain_aws`パッケージには、AWS公式のインテグレーションが用意されています。

```bash
pip install langchain-aws
```

`langchain_community`パッケージには、`boto3`の任意の依存関係を持つコミュニティインテグレーションもあります。

```bash
pip install langchain-community boto3
```

## LLMs

### Bedrock

>[Amazon Bedrock](https://aws.amazon.com/bedrock/)は、`AI21 Labs`、`Anthropic`、`Cohere`、`Meta`、`Stability AI`、`Amazon`などの
>主要なAI企業の高性能な基礎モデル(FM)を単一のAPIから選択できる完全管理型のサービスです。また、セキュリティ、プライバシー、責任あるAIのための広範な機能も提供しています。`Amazon Bedrock`を使えば、ユースケースに合わせてトップFMを簡単に試験・評価でき、ファインチューニングや`Retrieval Augmented Generation`(`RAG`)などの手法を使ってデータでカスタマイズし、エンタープライズシステムやデータソースを活用するエージェントを構築できます。`Amazon Bedrock`はサーバーレスなので、インフラの管理は不要で、AWS製品を使ってジェネレーティブAI機能を安全に統合・展開できます。

[使用例](/docs/integrations/llms/bedrock)をご覧ください。

```python
from langchain_aws import BedrockLLM
```

### Amazon API Gateway

>[Amazon API Gateway](https://aws.amazon.com/api-gateway/)は、開発者が簡単にAPIを作成、公開、管理、監視、セキュリティ保護できる完全管理型のサービスです。APIは、バックエンドサービスのデータ、ビジネスロジック、機能にアクセスするための"玄関口"の役割を果たします。`API Gateway`を使えば、リアルタイムの双方向通信アプリケーションを可能にするRESTfulAPIやWebSocketAPIを作成できます。`API Gateway`は、コンテナ化されたワークロードやサーバーレスワークロード、Webアプリケーションをサポートしています。

`API Gateway`は、数十万もの同時APIコールを受け付けて処理するための全てのタスクを処理します。これには、トラフィック管理、CORS対応、認証・アクセス制御、スロットリング、監視、APIバージョン管理などが含まれます。`API Gateway`には最小料金や初期費用はなく、受信したAPIコールと転送されたデータ量に応じて課金されます。`API Gateway`の段階的な料金モデルにより、APIの使用量が増えるにつれてコストを削減できます。

[使用例](/docs/integrations/llms/amazon_api_gateway)をご覧ください。

### Amazon DocumentDB ベクトル検索

>[Amazon DocumentDB (MongoDB 互換性付き)](https://docs.aws.amazon.com/documentdb/)は、クラウド上でMongoDB互換のデータベースを簡単に設定、運用、スケーリングできるようにします。
>Amazon DocumentDBを使えば、MongoDB で使用するのと同じアプリケーションコードやドライバー、ツールを使うことができます。
>Amazon DocumentDBのベクトル検索は、JSONベースのドキュメントデータベースの柔軟性と豊富な検索機能に、ベクトル検索の力を組み合わせたものです。

#### インストールとセットアップ

[詳細な設定手順](/docs/integrations/vectorstores/documentdb)を参照してください。

`pymongo` Pythonパッケージをインストールする必要があります。

```bash

```

#### AWS上でDocumentDBをデプロイする

AWSは、コンピューティング、データベース、ストレージ、分析など、さまざまな機能を提供するサービスを提供しています。AWSサービス全般の概要については、[Amazon Web Services によるクラウドコンピューティング](https://aws.amazon.com/what-is-aws/)を参照してください。

## リトリーバー

### Amazon Kendra

>[Amazon Kendra](https://docs.aws.amazon.com/kendra/latest/dg/what-is-kendra.html)は、`Amazon Web Services`(`AWS`)が提供する高度な自然言語処理(NLP)とマシンラーニングアルゴリズムを活用した高度な検索サービスです。
>`Kendra`は、組織内の様々なデータソースにわたって、ユーザーが必要な情報を素早く正確に見つけられるよう設計されており、生産性と意思決定の向上に役立ちます。

>`Kendra`を使えば、ドキュメント、FAQ、ナレッジベース、マニュアル、Webサイトなど、さまざまなコンテンツタイプを検索できます。複数の言語に対応し、複雑なクエリ、同義語、文脈的な意味を理解して、高い関連性の検索結果を提供します。

`langchain-aws`ライブラリをインストールする必要があります。

```bash
pip install langchain-aws
```

[使用例](/docs/integrations/retrievers/amazon_kendra_retriever)を確認してください。

```python
from langchain_aws import AmazonKendraRetriever
```

### AWS Lambda

>[`Amazon AWS Lambda`](https://aws.amazon.com/pm/lambda/)は、`Amazon Web Services`(`AWS`)が提供するサーバーレスコンピューティングサービスです。
>サーバーのプロビジョニングや管理を行うことなく、アプリケーションやサービスを構築・実行できます。このサーバーレスアーキテクチャにより、コードの記述とデプロイに集中でき、アプリケーションを実行するためのインフラストラクチャの拡張、パッチ適用、管理をAWSが自動的に行います。

`boto3` Pythonライブラリをインストールする必要があります。

```bash
pip install boto3
```

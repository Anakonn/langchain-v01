# チャットモデル

## 機能 (ネイティブでサポートされている)

すべての ChatModels は Runnable インターフェースを実装しており、すべてのメソッド (つまり `ainvoke`、`batch`、`abatch`、`stream`、`astream`) のデフォルトの実装が用意されています。これにより、すべての ChatModels に基本的な非同期、ストリーミング、バッチのサポートが提供されます。デフォルトでは以下のように実装されています:
- *非同期*サポートは、それぞれの同期メソッドを asyncio のデフォルトのスレッドプールエグゼキューターで呼び出すことになっています。これにより、ChatModel の実行中に、アプリケーションの他の非同期関数が進行できるようになります。
- *ストリーミング*サポートは、基礎となる ChatModel プロバイダーが返す最終結果を含む単一の値の `Iterator` (または非同期ストリーミングの場合は `AsyncIterator`) を返すことになっています。これは、トークンごとのストリーミングをサポートしていませんが、トークンの反復子を期待するコードが、すべての ChatModel インテグレーションで動作するようになります。
- *バッチ*サポートは、スレッドプールエグゼキューター (同期バッチの場合) または `asyncio.gather` (非同期バッチの場合) を使って、各入力を並行して基礎となる ChatModel に呼び出すことになっています。並行性は `RunnableConfig` の `max_concurrency` キーで制御できます。

各 ChatModel インテグレーションは、非同期やストリーミングを真に有効にするためのネイティブの実装を提供することができます。

# LangChain デコレーター ✨

~~~
免責事項: `LangChain デコレーター` は LangChain チームによって作成されたものではなく、LangChain チームによってサポートされているものでもありません。
~~~

>`LangChain デコレーター` は LangChain の上に構築されたレイヤーで、カスタムの LangChain プロンプトやチェーンを書くための構文糖衣 🍭 を提供します。
>
>フィードバック、問題、コントリビューションについては、こちらで issue を作成してください:
>[ju-bezdek/langchain-decorators](https://github.com/ju-bezdek/langchain-decorators)

主な原則と利点:

- より `Pythonic` な書き方
- インデントで中断されることのない、複数行のプロンプトを書くことができる
- IDE の組み込みサポートを活用して、**ヒント表示**、**型チェック**、**ドキュメントポップアップ**を使って、素早くプロンプトやパラメーターを確認できる
- 🦜🔗 LangChain エコシステムの全ての機能を活用できる
- **オプションのパラメーター**をサポートする
- プロンプト間でパラメーターを共有できるようにクラスにバインドできる

**LangChain デコレーター ✨** を使ったシンプルな例は以下の通りです。

```python

@llm_prompt
def write_me_short_post(topic:str, platform:str="twitter", audience:str = "developers")->str:
    """
    Write me a short header for my post about {topic} for {platform} platform.
    It should be for {audience} audience.
    (Max 15 words)
    """
    return

# run it naturally
write_me_short_post(topic="starwars")
# or
write_me_short_post(topic="starwars", platform="redit")
```

# その他のパラメーターの定義

ここでは、関数に `llm_prompt` デコレーターを付けることで、それを LLMChain として扱えるようにしています。実行するには

標準の LLMChain は、単に inputs_variables と prompt だけではなく、もっと多くのパラメーターを必要とします。この実装の詳細はデコレーターに隠されています。
動作の仕組みは以下の通りです:

1. **グローバル設定**を使う:

```python
# define global settings for all prompty (if not set - chatGPT is the current default)
from langchain_decorators import GlobalSettings

GlobalSettings.define_settings(
    default_llm=ChatOpenAI(temperature=0.0), this is default... can change it here globally
    default_streaming_llm=ChatOpenAI(temperature=0.0,streaming=True), this is default... can change it here for all ... will be used for streaming
)
```

2. 事前定義された **プロンプトタイプ**を使う

```python
#You can change the default prompt types
from langchain_decorators import PromptTypes, PromptTypeSettings

PromptTypes.AGENT_REASONING.llm = ChatOpenAI()

# Or you can just define your own ones:
class MyCustomPromptTypes(PromptTypes):
    GPT4=PromptTypeSettings(llm=ChatOpenAI(model="gpt-4"))

@llm_prompt(prompt_type=MyCustomPromptTypes.GPT4)
def write_a_complicated_code(app_idea:str)->str:
    ...

```

3. デコレーターの中で **設定を直接定義する**

```python

# ストリーミングの簡略化

ストリーミングを活用したい場合:
 - プロンプトを非同期関数として定義する必要があります
 - デコレーターでストリーミングをオンにするか、ストリーミング対応のPromptTypeを定義する
 - StreamingContextを使ってストリームをキャプチャする

このようにして、どのプロンプトをストリーミングすべきかを指定するだけで、LLMの選択やストリーミングハンドラーの作成・配布を気にする必要がありません。ストリーミングのオン/オフを切り替えるだけです。

ストリーミングは、ストリーミングコンテキスト内で呼び出された場合にのみ発生します。そこでは、ストリームを処理する簡単な関数を定義できます。

```python
# this code example is complete and should run as it is

from langchain_decorators import StreamingContext, llm_prompt

# this will mark the prompt for streaming (useful if we want stream just some prompts in our app... but don't want to pass distribute the callback handlers)
# note that only async functions can be streamed (will get an error if it's not)
@llm_prompt(capture_stream=True)
async def write_me_short_post(topic:str, platform:str="twitter", audience:str = "developers"):
    """
    Write me a short header for my post about {topic} for {platform} platform.
    It should be for {audience} audience.
    (Max 15 words)
    """
    pass



# just an arbitrary  function to demonstrate the streaming... will be some websockets code in the real world
tokens=[]
def capture_stream_func(new_token:str):
    tokens.append(new_token)

# if we want to capture the stream, we need to wrap the execution into StreamingContext...
# this will allow us to capture the stream even if the prompt call is hidden inside higher level method
# only the prompts marked with capture_stream will be captured here
with StreamingContext(stream_to_stdout=True, callback=capture_stream_func):
    result = await run_prompt()
    print("Stream finished ... we can distinguish tokens thanks to alternating colors")


print("\nWe've captured",len(tokens),"tokens🎉\n")
print("Here is the result:")
print(result)
```

# プロンプトの宣言

デフォルトでは、プロンプトは関数のドキュメント全体になりますが、プロンプトを明示的に指定することもできます。

## プロンプトのドキュメント化

ドキュメントの一部がプロンプト定義であることを指定するには、`<prompt>` 言語タグを使ったコードブロックを使います。

```python
@llm_prompt
def write_me_short_post(topic:str, platform:str="twitter", audience:str = "developers"):
    """
    Here is a good way to write a prompt as part of a function docstring, with additional documentation for devs.

    It needs to be a code block, marked as a `<prompt>` language
    ```<prompt>
    Write me a short header for my post about {topic} for {platform} platform.
    It should be for {audience} audience.
    (Max 15 words)
    ```

    Now only to code block above will be used as a prompt, and the rest of the docstring will be used as a description for developers.
    (It has also a nice benefit that IDE (like VS code) will display the prompt properly (not trying to parse it as markdown, and thus not showing new lines properly))
    """
    return
```

## チャットメッセージプロンプト

チャットモデルでは、プロンプトをメッセージテンプレートのセットとして定義するのが非常に便利です。こうするには:

```python
@llm_prompt
def simulate_conversation(human_input:str, agent_role:str="a pirate"):
    """
    ## System message
     - note the `:system` sufix inside the <prompt:_role_> tag


    ```<prompt:system>
    You are a {agent_role} hacker. You mus act like one.
    You reply always in code, using python or javascript code block...
    for example:

    ... do not reply with anything else.. just with code - respecting your role.
    ```

    # human message
    (we are using the real role that are enforced by the LLM - GPT supports system, assistant, user)
    ``` <prompt:user>
    Helo, who are you
    ```
    a reply:


    ``` <prompt:assistant>
    \``` python <<- escaping inner code block with \ that should be part of the prompt
    def hello():
        print("Argh... hello you pesky pirate")
    \```
    ```

    we can also add some history using placeholder
    ```<prompt:placeholder>
    {history}
    ```
    ```<prompt:user>
    {human_input}
    ```

    Now only to code block above will be used as a prompt, and the rest of the docstring will be used as a description for developers.
    (It has also a nice benefit that IDE (like VS code) will display the prompt properly (not trying to parse it as markdown, and thus not showing new lines properly))
    """
    pass

```

ここでのロールは、chatGPTなどのモデルネイティブのロール (assistant、user、system) です。

# オプションのセクション

- プロンプトの一部を完全にオプションとして定義できます
- そのセクションに入力がない場合、そのセクション全体がレンダリングされません

構文は以下の通りです:

```python
@llm_prompt
def prompt_with_optional_partials():
    """
    this text will be rendered always, but

    {? anything inside this block will be rendered only if all the {value}s parameters are not empty (None | "")   ?}

    you can also place it in between the words
    this too will be rendered{? , but
        this  block will be rendered only if {this_value} and {this_value}
        is not empty?} !
    """
```

## より複雑な構造

辞書/Pydantic の場合は、フォーマッティング命令を指定する必要があります...
これは面倒な作業になるため、出力パーサーにモデル (Pydantic) に基づいて命令を生成させることができます。

```python
from langchain_decorators import llm_prompt
from pydantic import BaseModel, Field


class TheOutputStructureWeExpect(BaseModel):
    name:str = Field (description="The name of the company")
    headline:str = Field( description="The description of the company (for landing page)")
    employees:list[str] = Field(description="5-8 fake employee names with their positions")

@llm_prompt()
def fake_company_generator(company_business:str)->TheOutputStructureWeExpect:
    """ Generate a fake company that {company_business}
    {FORMAT_INSTRUCTIONS}
    """
    return

company = fake_company_generator(company_business="sells cookies")

# print the result nicely formatted
print("Company name: ",company.name)
print("company headline: ",company.headline)
print("company employees: ",company.employees)

```

# プロンプトをオブジェクトにバインドする

```python
from pydantic import BaseModel
from langchain_decorators import llm_prompt

class AssistantPersonality(BaseModel):
    assistant_name:str
    assistant_role:str
    field:str

    @property
    def a_property(self):
        return "whatever"

    def hello_world(self, function_kwarg:str=None):
        """
        We can reference any {field} or {a_property} inside our prompt... and combine it with {function_kwarg} in the method
        """


    @llm_prompt
    def introduce_your_self(self)->str:
        """
        ``` <prompt:system>
        You are an assistant named {assistant_name}.
        Your role is to act as {assistant_role}
        ```
        ```<prompt:user>
        Introduce your self (in less than 20 words)
        ```
        """



personality = AssistantPersonality(assistant_name="John", assistant_role="a pirate")

print(personality.introduce_your_self(personality))
```

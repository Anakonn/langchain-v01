# 非構造化

> [Unstructured.IO](https://www.unstructured.io/)の`unstructured`パッケージは、PDFやWord文書などの生のソース文書から、きれいなテキストを抽出します。
このページでは、LangChainの中で[`unstructured`](https://github.com/Unstructured-IO/unstructured)エコシステムを使う方法を説明します。

## インストールとセットアップ

ローカルで実行されるローダーを使用している場合は、以下の手順で`unstructured`とその依存関係をローカルで実行できるようにします。

- `pip install unstructured`でPythonSDKをインストールします。
    - 文書固有の依存関係をextrasでインストールできます。例: `pip install "unstructured[docx]"`。
    - すべての文書タイプの依存関係をインストールするには、`pip install "unstructured[all-docs]"`を使います。
- お使いのシステムにまだ利用可能でない場合は、以下のシステム依存関係をインストールします。
  解析する文書タイプによっては、これらすべてが必要ではない可能性があります。
    - `libmagic-dev` (ファイルタイプ検出)
    - `poppler-utils` (画像とPDF)
    - `tesseract-ocr`(画像とPDF)
    - `libreoffice` (MS Officeドキュメント)
    - `pandoc` (EPUB)

セットアップが少なくて済むようにしたい場合は、単に`pip install unstructured`を実行し、`UnstructuredAPIFileLoader`または`UnstructuredAPIFileIOLoader`を使うことができます。
これにより、ホストされているUnstructuredAPIを使ってドキュメントを処理できます。

`Unstructured API`を使うには、APIキーが必要です。
[ここ](https://unstructured.io/api-key-hosted)でAPIキーを要求し、今すぐ使い始めることができます!
READMEの[ここ](https://github.com/Unstructured-IO/unstructured-api)を確認して、APIコールの使い始め方を確認してください。
フィードバックをお待ちしています。[コミュニティSlack](https://join.slack.com/t/unstructuredw-kbe4326/shared_invite/zt-1x7cgo0pg-PTptXWylzPQF9xZolzCnwQ)で進捗をお知らせください。
品質とパフォーマンスの向上に向けて、さらなる改善に取り組んでいます!
Unstructured APIをセルフホストしたり、ローカルで実行したい場合は、[ここ](https://github.com/Unstructured-IO/unstructured-api#dizzy-instructions-for-using-the-docker-image)の手順を確認してください。

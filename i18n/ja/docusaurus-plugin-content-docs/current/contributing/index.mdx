# 貢献者の皆さまへ

こんにちは! LangChainへの貢献にご興味をお持ちいただき、ありがとうございます。
急速に発展しているオープンソースプロジェクトとして、私たちは新機能、インフラの改善、ドキュメンテーションの向上、バグ修正など、あらゆる貢献を歓迎しています。

## 🗺️ ガイドライン

### 👩‍💻 貢献方法

LangChainに貢献する方法はさまざまです。一般的な貢献方法は以下の通りです:

- [**ドキュメンテーション**](/docs/contributing/documentation/style_guide): ドキュメントの改善にご協力ください。
- [**コード**](./code.mdx): コードの記述、バグ修正、インフラの改善にご協力ください。
- [**インテグレーション**](integrations.mdx): お気に入りのベンダーやツールとの連携にご協力ください。
- [**ディスカッション**](https://github.com/langchain-ai/langchain/discussions): ユーザーの質問に答えたり、ユーザーとの議論に参加したりしてください。

### 🚩 GitHub Issues

[issues](https://github.com/langchain-ai/langchain/issues)ページには、バグ、改善点、機能リクエストが掲載されています。

整理と発見のために、ラベルが付けられています。これらを活用してください。

課題に取り組む場合は、自分に割り当ててください。

新しい課題を追加する場合は、1つの問題に焦点を当ててください。
関連する課題がある場合は、それらをリンクさせてください。

私たちは可能な限りこれらの課題を最新の状態に保ちますが、この分野の急速な発展により、一部が古くなる可能性があります。
そのような場合は、ご連絡ください。

### 💭 GitHub Discussions

[discussions](https://github.com/langchain-ai/langchain/discussions)ページでは、ユーザーが使用方法の質問、設計上の決定、新機能の提案を行うことができます。

質問に答えることができる場合は、ぜひご協力ください。これにより、メンテナーがより多くの時間を開発とバグ修正に費やすことができます。

### 🙋 ヘルプの受け方

私たちの目標は、開発者の設定を可能な限り簡単にすることです。
設定に困難を感じた場合は、メンテナーにご連絡ください。
あなたの問題を解決するだけでなく、今後の貢献者のためにも、プロセスを改善したいと考えています。

同様に、私たちはコードベースの一定のリンティング、フォーマット、ドキュメンテーションの基準を設けています。
これらが面倒(あるいは煩わしい)と感じる場合は、メンテナーに連絡してヘルプを求めてください。
これらの基準がコードの提供の障害にならないようにしたいと考えています。

# 🌟 認知度

あなたの貢献がリリースに組み込まれた場合、Twitterで紹介させていただきます(ご希望の場合のみ)。
Twitterアカウントをお持ちの場合は、PRまたはその他の方法でお知らせください。

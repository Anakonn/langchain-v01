# 文字列評価器

文字列評価器は、LangChainの中で設計された、言語モデルの性能を評価するためのコンポーネントです。これは、生成された出力(予測)を参照文字列または入力と比較することで行われます。この比較は、言語モデルの精度や品質を測る上で重要なステップです。

実際の使用では、文字列評価器は、質問やプロンプトなどの入力に対して予測された文字列を評価するために使用されます。多くの場合、正しい応答または理想的な応答を定義するための参照ラベルまたはコンテキスト文字列が提供されます。これらの評価器は、アプリケーションの特定の要件に合わせてカスタマイズできます。

カスタムの文字列評価器を作成するには、`StringEvaluator`クラスを継承し、`_evaluate_strings`メソッドを実装します。非同期サポートが必要な場合は、`_aevaluate_strings`メソッドも実装します。

文字列評価器に関連する主要な属性とメソッドは以下の通りです:

- `evaluation_name`: 評価の名称を指定します。
- `requires_input`: 評価器が入力文字列を必要とするかどうかを示すブール値属性です。True の場合、入力が提供されていないと例外が発生します。False の場合、入力が提供されていると警告がログに記録されますが、評価には考慮されません。
- `requires_reference`: 評価器が参照ラベルを必要とするかどうかを示すブール値属性です。True の場合、参照が提供されていないと例外が発生します。False の場合、参照が提供されていると警告がログに記録されますが、評価には考慮されません。

文字列評価器はまた、以下のメソッドも実装しています:

- `aevaluate_strings`: 入力とラベルを任意で指定して、非同期的にチェーンまたは言語モデルの出力を評価します。
- `evaluate_strings`: 入力とラベルを任意で指定して、同期的にチェーンまたは言語モデルの出力を評価します。

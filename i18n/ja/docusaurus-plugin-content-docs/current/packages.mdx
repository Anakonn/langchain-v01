# 📕 パッケージバージョン管理

現在、LangChainには ad hoc なリリースプロセスがあります。リリースは高頻度でメンテナーによって行われ、[PyPI](https://pypi.org/)に公開されています。

さまざまなパッケージのバージョン管理は少し異なります。

## `langchain-core`

`langchain-core`は現在バージョン `0.1.x`です。

`langchain-core`には、LangChain生態系全体の基本的な抽象化とランタイムが含まれているため、破壊的な変更については事前に通知し、バージョンアップを行います。ただし、`beta`デコレーターでマークされた機能は例外です(API リファレンスで確認でき、そのような機能を使用するときに警告が表示されます)。ベータ機能を提供する理由は、この分野の変化のスピードが速いため、素早く進めることが優先されるためです。

マイナーバージョンアップは以下の場合に行われます:

- `beta`としてマークされた公開インターフェイスの破壊的な変更

パッチバージョンアップは以下の場合に行われます:

- バグ修正
- 新機能の追加
- プライベートインターフェイスの変更
- `beta`機能の変更

## `langchain`

`langchain`は現在バージョン `0.1.x`です。

マイナーバージョンアップは以下の場合に行われます:

- `beta`としてマークされていない公開インターフェイスの破壊的な変更

パッチバージョンアップは以下の場合に行われます:

- バグ修正
- 新機能の追加
- プライベートインターフェイスの変更
- `beta`機能の変更

`langchain` v0.2リリースに向けて作業中で、一部の従来のChainやAgentに対する破壊的な変更が含まれる予定です。

また、`langchain-community`の依存関係を削除し、`langchain-community`に移動したインテグレーションの再エクスポートを停止します。

## `langchain-community`

`langchain-community`は現在バージョン `0.0.x`です。

すべての変更にはパッチバージョンアップが伴います。

## `langchain-experimental`

`langchain-experimental`は現在バージョン `0.0.x`です。

すべての変更にはパッチバージョンアップが伴います。

## パートナーパッケージ

パートナーパッケージは独立してバージョン管理されています。

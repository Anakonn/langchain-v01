# メッセージタイプ

ChatModelsは入力としてメッセージのリストを受け取り、メッセージを返します。メッセージには幾つかの異なるタイプがあります。すべてのメッセージには `role` と `content` プロパティがあります。 `role` はメッセージを発しているWHOを表します。LangChainには役割ごとに異なるメッセージクラスがあります。 `content` プロパティはメッセージの内容を表します。これには以下のようなものがあります:

- 文字列 (ほとんどのモデルがこのタイプのコンテンツを扱う)
- 辞書のリスト (これはマルチモーダル入力で使用され、辞書にはその入力タイプと入力場所に関する情報が含まれる)

さらに、メッセージには `additional_kwargs` プロパティがあります。ここには、プロバイダ固有の入力パラメータなど、一般的ではないメッセージに関する追加情報を渡すことができます。最も有名な例は OpenAI の `function_call` です。

### HumanMessage

これはユーザーからのメッセージを表します。一般的にはコンテンツのみで構成されます。

### AIMessage

これはモデルからのメッセージを表します。 OpenAI のツールコールなどの `additional_kwargs` を持つ場合があります。

### SystemMessage

これはシステムメッセージを表し、モデルの動作方法を指示します。一般的にはコンテンツのみで構成されます。すべてのモデルがこれをサポートしているわけではありません。

### FunctionMessage

これは関数呼び出しの結果を表します。 `role` と `content` に加えて、この結果を生成した関数の名前を表す `name` パラメータがあります。

### ToolMessage

これはツールコールの結果を表します。これは OpenAI の `function` と `tool` メッセージタイプに合わせるために、FunctionMessageとは区別されています。 `role` と `content` に加えて、この結果を生成したツールコールのIDを表す `tool_call_id` パラメータがあります。

# エージェントのタイプ

これは、いくつかの次元に沿って利用可能なすべてのエージェントを分類するものです。

**対象モデルタイプ**

このエージェントがチャットモデル(メッセージを受け取り、メッセージを出力する)またはLLM(文字列を受け取り、文字列を出力する)を対象としているかどうかを示します。これが影響するのは主にプロンプト戦略です。対象とは異なるタイプのモデルでエージェントを使用することはできますが、同じ品質の結果が得られない可能性があります。

**チャット履歴のサポート**

これらのエージェントタイプがチャット履歴をサポートするかどうかを示します。サポートする場合、それはチャットボットとして使用できることを意味します。サポートしない場合は、単一のタスクに適しています。チャット履歴のサポートには一般により優れたモデルが必要なため、より低性能なモデルを対象とした初期のエージェントタイプではサポートされていない可能性があります。

**複数入力ツールのサポート**

これらのエージェントタイプが複数の入力を持つツールをサポートするかどうかを示します。ツールが単一の入力しか必要としない場合、LLMがどのように呼び出せばよいかを知るのは一般的に簡単です。したがって、より低性能なモデルを対象とした初期のエージェントタイプの多くはこれをサポートしていない可能性があります。

**並列関数呼び出しのサポート**

LLMが複数のツールを同時に呼び出すことができると、エージェントのタスクを大幅に高速化できます。ただし、LLMがこれを行うのは非常に難しいため、一部のエージェントタイプではサポートされていません。

**必要なモデルパラメータ**

このエージェントが追加のパラメータをモデルにサポートさせる必要があるかどうかを示します。一部のエージェントタイプは、OpenAI の関数呼び出しのようなものを利用しており、それには他のモデルパラメータが必要です。必要なパラメータがない場合は、すべてがプロンプティングによって行われることを意味します。

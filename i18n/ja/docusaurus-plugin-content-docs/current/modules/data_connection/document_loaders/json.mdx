# JSON

>[JSON (JavaScript Object Notation)](https://en.wikipedia.org/wiki/JSON) は、属性–値ペアおよび配列（または他のシリアライズ可能な値）からなるデータオブジェクトを保存および送信するために、人間が読めるテキストを使用するオープン標準ファイル形式およびデータ交換形式です。

>[JSON Lines](https://jsonlines.org/) は、各行が有効なJSON値であるファイル形式です。

>`JSONLoader` は、指定された [jq スキーマ](https://en.wikipedia.org/wiki/Jq_(programming_language)) を使用してJSONファイルを解析します。これは `jq` Pythonパッケージを使用します。
`jq` 構文の詳細なドキュメントについては、この [マニュアル](https://stedolan.github.io/jq/manual/#Basicfilters) を確認してください。

## `JSONLoader` の使用

### JSON ファイル

```python
loader = JSONLoader(
    file_path='./example_data/facebook_chat.json',
    jq_schema='.messages[].content',
    text_content=False)

data = loader.load()
```

### jq スキーマ `content_key` を使用したJSONファイル

jqスキーマ内のcontent_keyを使用してJSONファイルからドキュメントをロードするには、is_content_key_jq_parsable=Trueを設定します。
content_keyが互換性があり、jqスキーマを使用して解析できることを確認してください。

```python
file_path = './sample.json'
pprint(Path(file_path).read_text())
```

```json
    {"data": [
        {"attributes": {
            "message": "message1",
            "tags": [
            "tag1"]},
        "id": "1"},
        {"attributes": {
            "message": "message2",
            "tags": [
            "tag2"]},
        "id": "2"}]}
```

```python
loader = JSONLoader(
    file_path=file_path,
    jq_schema=".data[]",
    content_key=".attributes.message",
    is_content_key_jq_parsable=True,
)

data = loader.load()
```


## メタデータの抽出

一般的に、作成するドキュメントにJSONファイル内のメタデータを含めたいと考えます。

以下は、`JSONLoader` を使用してメタデータを抽出する方法を示しています。

いくつかの重要な変更点に注意する必要があります。前の例ではメタデータを収集しなかったため、スキーマ内で `page_content` の値を直接抽出する場所を指定できました。

```python
.messages[].content
```

現在の例では、`messages` フィールド内のレコードを反復処理するようにローダーに指示する必要があります。jqスキーマは次のようになります:

```python
.messages[]
```

これにより、レコード（辞書）を `metadata_func` に渡すことができるようになります。`metadata_func` は、最終的な `Document` オブジェクトに保存されるメタデータに含めるべき情報を特定する役割を担っています。

さらに、ローダー内で `content_key` 引数を介して、`page_content` の値を抽出するレコードのキーを明示的に指定する必要があります。

```python
# Define the metadata extraction function.
def metadata_func(record: dict, metadata: dict) -> dict:

    metadata["sender_name"] = record.get("sender_name")
    metadata["timestamp_ms"] = record.get("timestamp_ms")

    return metadata


loader = JSONLoader(
    file_path='./example_data/facebook_chat.json',
    jq_schema='.messages[]',
    content_key="content",
    metadata_func=metadata_func
)

data = loader.load()
```

## `metadata_func`

上記のように、`metadata_func` は `JSONLoader` によって生成されたデフォルトのメタデータを受け入れます。これにより、メタデータの形式に関してユーザーに完全な制御が与えられます。

例えば、デフォルトのメタデータには `source` および `seq_num` キーが含まれます。しかし、JSONデータにもこれらのキーが含まれている可能性があります。ユーザーは `metadata_func` を利用してデフォルトのキー名を変更し、JSONデータからのキーを使用することができます。

以下の例では、`source` を変更して `langchain` ディレクトリに相対的なファイルソースの情報のみを含める方法を示しています。

```python
# Define the metadata extraction function.
def metadata_func(record: dict, metadata: dict) -> dict:

    metadata["sender_name"] = record.get("sender_name")
    metadata["timestamp_ms"] = record.get("timestamp_ms")

    if "source" in metadata:
        source = metadata["source"].split("/")
        source = source[source.index("langchain"):]
        metadata["source"] = "/".join(source)

    return metadata


loader = JSONLoader(
    file_path='./example_data/facebook_chat.json',
    jq_schema='.messages[]',
    content_key="content",
    metadata_func=metadata_func
)

data = loader.load()
```

## jq スキーマを使用した一般的なJSON構造

以下のリストは、構造に応じてJSONデータからコンテンツを抽出するためにユーザーが使用できる可能な `jq_schema` の参考を提供します。

```output
JSON        -> [{"text": ...}, {"text": ...}, {"text": ...}]
jq_schema   -> ".[].text"

JSON        -> {"key": [{"text": ...}, {"text": ...}, {"text": ...}]}
jq_schema   -> ".key[].text"

JSON        -> ["...", "...", "..."]
jq_schema   -> ".[]"
```

# 検索

多くのLLMアプリケーションでは、モデルの学習データに含まれていないユーザー固有のデータが必要とされます。
これを実現する主な方法は、Retrieval Augmented Generation (RAG)です。
このプロセスでは、外部データが*検索*され、その後*生成*の際にLLMに渡されます。

LangChainは、RAGアプリケーションのためのすべての構成要素を提供しています - 簡単なものから複雑なものまで。
このドキュメントのこのセクションでは、*検索*の段階 - つまりデータの取得 - に関するすべてを扱います。
これは単純に見えますが、微妙に複雑になることがあります。
これには、いくつかの重要なモジュールが含まれています。

![データ接続プロセスを示す図解図: ソース、ロード、変換、埋め込み、ストア、検索の各ステップ。](/img/data_connection.jpg "データ接続プロセス図")

## [ドキュメントローダー](/docs/modules/data_connection/document_loaders/)

**ドキュメントローダー**は、さまざまなソースからドキュメントを読み込みます。
LangChainは100以上のドキュメントローダーを提供するほか、AirByteやUnstructuredなどの主要プロバイダとの統合も行っています。
LangChainは、あらゆるタイプのドキュメント(HTML、PDF、コードなど)を、あらゆる場所(プライベートS3バケット、公開Webサイトなど)から読み込むための統合を提供しています。

## [テキスト分割](/docs/modules/data_connection/document_transformers/)

検索の重要な部分は、関連する部分のみをドキュメントから取り出すことです。
これには、ドキュメントを検索に適したものに準備するための変換ステップが複数含まれます。
その主要なものの1つが、大きなドキュメントを小さな塊(チャンク)に分割することです。
LangChainは、この処理を行うための複数の変換アルゴリズムを提供しており、特定のドキュメントタイプ(コード、Markdown など)に最適化されたロジックも用意しています。

## [テキスト埋め込みモデル](/docs/modules/data_connection/text_embedding/)

検索の別の重要な部分は、ドキュメントの埋め込みを作成することです。
埋め込みは、テキストの意味的な意味を捉えており、類似したテキストの部分を素早く効率的に見つけることができます。
LangChainは、オープンソースからプロプライエタリなAPIまで、25種類以上の埋め込みプロバイダと手法との統合を提供しており、ニーズに最適なものを選択できます。
LangChainは標準のインターフェイスを提供しているため、モデル間の簡単な切り替えが可能です。

## [ベクトルストア](/docs/modules/data_connection/vectorstores/)

埋め込みの台頭に伴い、これらの埋め込みを効率的に保存および検索できるデータベースの需要が高まっています。
LangChainは、オープンソースのローカルなものからクラウドホスティングのプロプライエタリなものまで、50種類以上のベクトルストアとの統合を提供しており、ニーズに最適なものを選択できます。
LangChainは標準のインターフェイスを公開しているため、ベクトルストア間の簡単な切り替えが可能です。

## [リトリーバー](/docs/modules/data_connection/retrievers/)

データがデータベースに格納されたら、それを検索する必要があります。
LangChainは多様な検索アルゴリズムをサポートしており、ここが最も付加価値を提供する部分の1つです。
LangChainは、簡単に始められる基本的な手法 - 単純な意味検索 - をサポートしています。
しかし、さらにパフォーマンスを高めるためのアルゴリズムも追加しています。
これらには以下が含まれます:

- [Parent Document Retriever](/docs/modules/data_connection/retrievers/parent_document_retriever): これにより、親ドキュメントごとに複数の埋め込みを作成できるため、小さな塊を検索しつつ、より大きなコンテキストを返すことができます。
- [Self Query Retriever](/docs/modules/data_connection/retrievers/self_query): ユーザーの質問には、単なる意味的なものだけでなく、メタデータフィルターとして表現できるような論理的なものが含まれることがあります。Self-queryは、質問の*意味的*な部分と*メタデータフィルター*を区別することができます。
- [Ensemble Retriever](/docs/modules/data_connection/retrievers/ensemble): 場合によっては、複数のソースからドキュメントを検索したり、複数の異なるアルゴリズムを使用したりする必要があります。Ensemble retrieverを使うと、これを簡単に行えます。
- その他

## [インデックス作成](/docs/modules/data_connection/indexing)

LangChainの**インデックス作成API**は、任意のソースからデータをベクトルストアに同期させ、以下を支援します:

- ベクトルストアへの重複コンテンツの書き込みの回避
- 変更されていないコンテンツの再書き込みの回避
- 変更されていないコンテンツの再埋め込みの回避

これらにより、時間とコストの節約、およびベクトル検索結果の改善が期待できます。

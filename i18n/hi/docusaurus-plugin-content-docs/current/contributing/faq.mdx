# अक्सर पूछे जाने वाले प्रश्न

## पुल अनुरोध (पीआर)

### मैं अपने पीआर में संरक्षक को संपादित करने की अनुमति कैसे दे सकता हूं?

जब आप एक पुल अनुरोध प्रस्तुत करते हैं, तो इसे विलय करने से पहले अतिरिक्त परिवर्तन करने की आवश्यकता हो सकती है। अक्सर, कोड समीक्षा में आपसे पूछने की बजाय, संरक्षकों के लिए इन परिवर्तनों को स्वयं करना अधिक कुशल होता है।

डिफ़ॉल्ट रूप से, अधिकांश पुल अनुरोधों में दाईं ओर के साइडबार में `✅ संरक्षक इस पुल अनुरोध को संपादित करने की अनुमति दी जाती है।` बैज होगा।

यदि आप इस बैज को नहीं देखते हैं, तो आप इस सेटिंग को अपने द्वारा पुल अनुरोध किए जा रहे फोर्क के लिए बंद कर सकते हैं। अधिक जानकारी के लिए [इस Github दस्तावेज़ पृष्ठ](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/allowing-changes-to-a-pull-request-branch-created-from-a-fork) देखें।

ध्यान दें कि Github **संगठनों** के लिए फोर्क के लिए इस सेटिंग को सक्षम करने की अनुमति नहीं देता है ([मुद्दा](https://github.com/orgs/community/discussions/5634))। यदि आप किसी संगठन में काम कर रहे हैं, तो हम आपको व्यक्तिगत फोर्क से अपना पीआर प्रस्तुत करने की सिफारिश करते हैं ताकि यह सेटिंग सक्षम हो सके।

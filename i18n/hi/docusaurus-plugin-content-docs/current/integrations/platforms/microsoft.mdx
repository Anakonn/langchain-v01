## चैट मॉडल

### एज़्योर ओपनएआई

>[माइक्रोसॉफ्ट एज़्योर](https://en.wikipedia.org/wiki/Microsoft_Azure), जिसे `एज़्योर` के रूप में भी जाना जाता है, एक क्लाउड कंप्यूटिंग प्लेटफॉर्म है जो `माइक्रोसॉफ्ट` द्वारा संचालित है, जो वैश्विक डेटा केंद्रों के माध्यम से एप्लिकेशन और सेवाओं की पहुंच, प्रबंधन और विकास प्रदान करता है। यह कई क्षमताओं का समर्थन करता है, जिनमें सॉफ्टवेयर के रूप में सेवा (SaaS), प्लेटफॉर्म के रूप में सेवा (PaaS) और बुनियादी ढांचे के रूप में सेवा (IaaS) शामिल हैं। `माइक्रोसॉफ्ट एज़्योर` कई प्रोग्रामिंग भाषाओं, उपकरणों और फ्रेमवर्क का समर्थन करता है, जिनमें माइक्रोसॉफ्ट-विशिष्ट और तृतीय-पक्ष सॉफ्टवेयर और प्रणाली शामिल हैं।

>[एज़्योर ओपनएआई](https://learn.microsoft.com/en-us/azure/cognitive-services/openai/) एक `एज़्योर` सेवा है जो `ओपनएआई` से शक्तिशाली भाषा मॉडल प्रदान करता है, जिनमें `जीपीटी-3`, `कोडेक्स` और `एम्बेडिंग मॉडल` श्रृंखला शामिल हैं जो सामग्री उत्पादन, सारांश, सेमेंटिक खोज और प्राकृतिक भाषा से कोड अनुवाद के लिए हैं।


### एज़्योर ब्लॉब स्टोरेज

>[एज़्योर ब्लॉब स्टोरेज](https://learn.microsoft.com/en-us/azure/storage/blobs/storage-blobs-introduction) माइक्रोसॉफ्ट का क्लाउड के लिए ऑब्जेक्ट स्टोरेज समाधान है। ब्लॉब स्टोरेज असंरचित डेटा की भारी मात्रा को संग्रहीत करने के लिए अनुकूलित है। असंरचित डेटा वह डेटा है जो किसी विशिष्ट डेटा मॉडल या परिभाषा का पालन नहीं करता है, जैसे पाठ या बाइनरी डेटा।

>[एज़्योर फ़ाइल्स](https://learn.microsoft.com/en-us/azure/storage/files/storage-files-introduction) उद्योग मानक सर्वर संदेश ब्लॉक (`एसएमबी`) प्रोटोकॉल, नेटवर्क फ़ाइल प्रणाली (`एनएफएस`) प्रोटोकॉल और `एज़्योर फ़ाइल्स आरईएसटी एपीआई` के माध्यम से पूरी तरह से प्रबंधित क्लाउड फ़ाइल साझा प्रदान करता है। `एज़्योर फ़ाइल्स` `एज़्योर ब्लॉब स्टोरेज` पर आधारित हैं।

`एज़्योर ब्लॉब स्टोरेज` निम्नलिखित के लिए डिज़ाइन किया गया है:
- ब्राउज़र के लिए सीधे छवियों या दस्तावेजों को सर्व करना।
- वितरित पहुंच के लिए फ़ाइलों को संग्रहीत करना।
- वीडियो और ऑडियो स्ट्रीमिंग।
- लॉग फ़ाइलों में लिखना।
- बैकअप और पुनर्स्थापना, आपदा प्रतिकार और संग्रहण के लिए डेटा संग्रहीत करना।
- ऑन-प्रीमाइसेस या एज़्योर-होस्टेड सेवा द्वारा विश्लेषण के लिए डेटा संग्रहीत करना।

```bash
pip install azure-storage-blob
```

### Azure Cosmos DB

>[Azure Cosmos DB for MongoDB vCore](https://learn.microsoft.com/en-us/azure/cosmos-db/mongodb/vcore/) पूर्ण मूल MongoDB समर्थन के साथ एक डेटाबेस बनाना आसान बनाता है।
> आप अपने MongoDB अनुभव को लागू कर सकते हैं और अपने पसंदीदा MongoDB ड्राइवर, SDK और उपकरणों का उपयोग करना जारी रख सकते हैं क्योंकि आप अपने एप्लिकेशन को MongoDB vCore खाते के कनेक्शन स्ट्रिंग पर इंगित करते हैं।
> Azure Cosmos DB for MongoDB vCore में वेक्टर खोज का उपयोग करें ताकि आप अपने AI-आधारित अनुप्रयोगों को Azure Cosmos DB में संग्रहीत अपने डेटा के साथ सुचारू रूप से एकीकृत कर सकें।

## रिट्रीवर

### Azure AI Search

>[Azure AI Search](https://learn.microsoft.com/en-us/azure/search/search-what-is-azure-search) (पहले `Azure Search` या `Azure Cognitive Search` के रूप में जाना जाता था) एक क्लाउड खोज सेवा है जो विकासकर्ताओं को वेब, मोबाइल और उद्यम अनुप्रयोगों में निजी, विषम सामग्री पर एक समृद्ध खोज अनुभव बनाने के लिए बुनियादी ढांचा, API और उपकरण प्रदान करती है।

>खोज किसी भी ऐसे ऐप के लिए आधारभूत है जो उपयोगकर्ताओं को पाठ प्रदर्शित करता है, जहां आम पृष्ठभूमि में कैटलॉग या दस्तावेज़ खोज, ऑनलाइन खुदरा अनुप्रयोग या स्वामित्व वाली सामग्री पर डेटा अन्वेषण शामिल हैं। जब आप एक खोज सेवा बनाते हैं, तो आप निम्नलिखित क्षमताओं के साथ काम करेंगे:
>- उपयोगकर्ता-स्वामित्व वाली सामग्री वाले एक खोज सूचकांक पर पूर्ण पाठ खोज के लिए एक खोज इंजन
>- समृद्ध अनुक्रमण, लेक्सिकल विश्लेषण और वैकल्पिक AI समृद्धि के साथ सामग्री निकालने और रूपांतरण के लिए
>- पाठ खोज, अस्पष्ट खोज, स्वतः पूरा करना, भू-खोज और अधिक के लिए समृद्ध क्वेरी वाक्यविन्यास
>- Azure SDK में REST API और क्लाइंट लाइब्रेरी के माध्यम से प्रोग्रामेबिलिटी
>- डेटा स्तर, मशीन लर्निंग स्तर और AI (AI सेवाएं) पर Azure एकीकरण

[सेटअप निर्देश](https://learn.microsoft.com/en-us/azure/search/search-create-service-portal) देखें।

## अधिक

### Microsoft Presidio

>[Presidio](https://microsoft.github.io/presidio/) (लैटिन प्रैसिडियम 'सुरक्षा, गार्निजन' से उत्पन्न)
> संवेदनशील डेटा के उचित प्रबंधन और शासन को सुनिश्चित करने में मदद करता है। यह पाठ और छवियों में निजी इकाइयों जैसे क्रेडिट कार्ड नंबर, नाम, स्थान, सामाजिक सुरक्षा नंबर, बिटकॉइन वॉलेट, यूएस फोन नंबर, वित्तीय डेटा और अधिक के लिए तेज पहचान और अनोनीमीकरण मॉड्यूल प्रदान करता है।

पहले, आपको कई पायथन पैकेज स्थापित करने और एक `SpaCy` मॉडल डाउनलोड करने की आवश्यकता है।

```bash
pip install langchain-experimental openai presidio-analyzer presidio-anonymizer spacy Faker
python -m spacy download en_core_web_lg
```

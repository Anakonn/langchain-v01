# पाठ एम्बेडिंग मॉडल

:::info
[एकीकरण](/docs/integrations/text_embedding/) पर दस्तावेज़ के लिए पाठ एम्बेडिंग मॉडल प्रदाताओं के साथ बनाए गए एकीकरणों के बारे में जानकारी प्राप्त करें।
:::

Embeddings क्लास पाठ एम्बेडिंग मॉडल के साथ इंटरफ़ेस करने के लिए डिज़ाइन की गई एक क्लास है। कई एम्बेडिंग मॉडल प्रदाता (OpenAI, Cohere, Hugging Face आदि) हैं - यह क्लास उनमें से सभी के लिए एक मानक इंटरफ़ेस प्रदान करने के लिए डिज़ाइन की गई है।

एम्बेडिंग्स पाठ के एक टुकड़े का वेक्टर प्रतिनिधित्व बनाते हैं। यह इसलिए उपयोगी है क्योंकि इसका मतलब है कि हम पाठ को वेक्टर स्पेस में सोच सकते हैं, और ऐसी चीजें कर सकते हैं जैसे सेमांटिक खोज जहां हम वेक्टर स्पेस में सबसे समान टुकड़ों को ढूंढते हैं।

LangChain में आधारभूत Embeddings क्लास दो विधियां प्रदान करती है: एक दस्तावेज़ों को एम्बेड करने के लिए और एक क्वेरी को एम्बेड करने के लिए। पहला कई पाठों को इनपुट के रूप में लेता है, जबकि दूसरा एक पाठ को लेता है। इन्हें दो अलग-अलग विधियों के रूप में रखने का कारण यह है कि कुछ एम्बेडिंग प्रदाताओं के पास दस्तावेज़ों (खोजे जाने वाले) और क्वेरी (खोज क्वेरी स्वयं) के लिए अलग-अलग एम्बेडिंग विधियां होती हैं।

```bash

```

यदि आप पर्यावरण चर सेट करना नहीं चाहते हैं, तो आप OpenAI LLM क्लास को प्रारंभ करते समय `api_key` नामित पैरामीटर के माध्यम से कुंजी को सीधे पास कर सकते हैं:

### `embed_documents`

#### पाठों की सूची को एम्बेड करना

```python
embeddings = embeddings_model.embed_documents(
    [
        "Hi there!",
        "Oh, hello!",
        "What's your name?",
        "My friends call me World",
        "Hello World!"
    ]
)
len(embeddings), len(embeddings[0])
```

```output
(5, 1536)
```

### `embed_query`

#### एकल क्वेरी को एम्बेड करना

अन्य एम्बेड किए गए पाठों के साथ तुलना करने के उद्देश्य से एक अकेले पाठ को एम्बेड करें।

```python
embedded_query = embeddings_model.embed_query("What was the name mentioned in the conversation?")
embedded_query[:5]
```

```output
[0.0053587136790156364,
 -0.0004999046213924885,
 0.038883671164512634,
 -0.003001077566295862,
 -0.00900818221271038]
```

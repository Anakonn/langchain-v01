# 채팅 모델

## 기능 (자체 지원)

모든 ChatModel은 Runnable 인터페이스를 구현하며, 이는 모든 메서드(예: `ainvoke`, `batch`, `abatch`, `stream`, `astream`)의 기본 구현을 제공합니다. 따라서 모든 ChatModel은 기본적으로 아래와 같이 비동기, 스트리밍 및 배치 지원을 갖추고 있습니다:
- *비동기* 지원은 기본적으로 asyncio의 기본 스레드 풀 실행자에서 해당 동기 메서드를 호출하는 것으로 설정됩니다. 이를 통해 ChatModel이 실행되는 동안 애플리케이션의 다른 비동기 함수들이 백그라운드 스레드에서 이 호출을 이동함으로써 진행될 수 있습니다.
- *스트리밍* 지원은 기본적으로 최종 결과를 반환하는 단일 값의 `Iterator`(비동기 스트리밍의 경우 `AsyncIterator`)를 반환하는 것으로 설정됩니다. 이는 ChatModel 제공자로부터 토큰별 스트리밍을 제공하지 않지만, 토큰의 iterator를 기대하는 코드가 우리의 모든 ChatModel 통합에 대해 작동할 수 있도록 보장합니다.
- *배치* 지원은 기본적으로 스레드 풀 실행자(동기 배치의 경우)를 사용하거나 `asyncio.gather`(비동기 배치의 경우)를 사용하여 각 입력에 대해 병렬로 기본 ChatModel을 호출하는 것으로 설정됩니다. 동시성은 `RunnableConfig`의 `max_concurrency` 키로 제어할 수 있습니다.

각 ChatModel 통합은 선택적으로 비동기 또는 스트리밍을 실제로 활성화하기 위한 기본 구현을 제공할 수 있습니다.

# 자주 묻는 질문

## 풀 리퀘스트 (PRs)

### 메인테이너가 내 PR을 수정하도록 허용하려면 어떻게 해야 하나요?

풀 리퀘스트를 제출할 때, 병합하기 전에 추가적인 변경이 필요할 수 있습니다. 종종 코드 리뷰에서 당신에게 요청하는 것보다 메인테이너가 직접 이러한 변경을 하고 병합하는 것이 더 효율적일 수 있습니다.

기본적으로 대부분의 풀 리퀘스트는 오른쪽 사이드바에 
`✅ Maintainers are allowed to edit this pull request.`
배지를 표시합니다.

이 배지가 보이지 않으면, 당신이 풀 리퀘스트를 요청하는 포크에 대해 이 설정이 꺼져 있을 수 있습니다. 자세한 내용은 [이 Github 문서 페이지](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/allowing-changes-to-a-pull-request-branch-created-from-a-fork)를 참조하세요.

특히, Github는 **조직**의 포크에 대해 이 설정을 활성화할 수 없도록 하고 있습니다 ([이슈](https://github.com/orgs/community/discussions/5634)).
조직에서 작업하는 경우, 이 설정을 활성화하기 위해 개인 포크에서 PR을 제출하는 것이 좋습니다.

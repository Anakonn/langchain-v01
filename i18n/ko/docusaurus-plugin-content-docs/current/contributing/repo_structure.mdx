# 리포지토리 구조

LangChain 코드 또는 문서에 기여할 계획이라면, 리포지토리의 고수준 구조를 이해하는 것이 유용할 수 있습니다.

LangChain은 여러 패키지를 포함하는 [모노레포](https://en.wikipedia.org/wiki/Monorepo)로 구성되어 있습니다.

다음은 트리 형태로 시각화한 구조입니다:

```text
.
├── cookbook # 튜토리얼 및 예제
├── docs # https://python.langchain.com/에 있는 문서의 콘텐츠
├── libs
│   ├── langchain # 메인 패키지
│   │   ├── tests/unit_tests # 유닛 테스트 (각 패키지에 존재하지만 간결성을 위해 생략)
│   │   ├── tests/integration_tests # 통합 테스트 (각 패키지에 존재하지만 간결성을 위해 생략)
│   ├── langchain-community # 서드 파티 통합
│   ├── langchain-core # 주요 추상화의 기본 인터페이스
│   ├── langchain-experimental # 실험적 구성 요소 및 체인
│   ├── partners
│       ├── langchain-partner-1
│       ├── langchain-partner-2
│       ├── ...
│
├── templates # 다양한 작업을 위한 쉽게 배포 가능한 참조 아키텍처 모음
```

루트 디렉터리에는 다음 파일도 포함되어 있습니다:

- `pyproject.toml`: 문서 빌드 및 린팅, 요리책 종속성.
- `Makefile`: 빌드, 린팅, 문서 및 요리책을 위한 단축키가 포함된 파일.

루트 디렉터리 수준에는 다른 파일들도 있지만, 그 존재는 자명할 것입니다. 자유롭게 둘러보세요!

## 문서

`/docs` 디렉터리에는 https://python.langchain.com/에서 표시되는 문서와 관련된 API 참조 https://api.python.langchain.com/en/latest/langchain_api_reference.html의 콘텐츠가 포함되어 있습니다.

문서에 기여하는 방법을 배우려면 [문서](/docs/contributing/documentation/style_guide) 가이드를 참조하세요.

## 코드

`/libs` 디렉터리에는 LangChain 패키지의 코드가 포함되어 있습니다.

코드 기여 방법에 대해 자세히 알아보려면 다음 가이드를 참조하세요:

- [코드](./code.mdx) LangChain 코드베이스에서 개발하는 방법을 배우세요.
- [통합](./integrations.mdx) langchain-community에 서드 파티 통합을 기여하거나 새로운 파트너 패키지를 시작하는 방법을 배우세요.
- [테스트](./testing.mdx) 패키지에 대한 테스트를 작성하는 방법을 배우기 위한 가이드라인을 확인하세요.

# 기여자 여러분 환영합니다

안녕하세요! LangChain에 기여에 관심을 가져주셔서 감사합니다.
빠르게 발전하는 분야의 오픈 소스 프로젝트로서, 새로운 기능, 인프라 개선, 문서 향상, 버그 수정 등 모든 기여를 환영합니다.

## 🗺️ 가이드라인

### 👩‍💻 기여 방법

LangChain에 기여할 수 있는 방법은 다양합니다. 사람들이 기여하는 일반적인 방법은 다음과 같습니다:

- [**문서**](/docs/contributing/documentation/style_guide): 우리의 문서, 이 문서를 포함하여, 개선하는 것을 도와주세요!
- [**코드**](./code.mdx): 코드 작성, 버그 수정, 인프라 개선을 도와주세요.
- [**통합**](integrations.mdx): 좋아하는 벤더와 도구와의 통합을 도와주세요.
- [**토론**](https://github.com/langchain-ai/langchain/discussions): 사용 질문에 답변하고 사용자와 문제를 논의하는 것을 도와주세요.

### 🚩 GitHub 이슈

우리의 [이슈](https://github.com/langchain-ai/langchain/issues) 페이지는 버그, 개선사항 및 기능 요청으로 업데이트됩니다.

관심 있는 이슈를 정렬하고 발견하는 데 도움이 되는 라벨 체계가 있습니다. 이를 사용하여 이슈를 조직화하세요.

이슈 작업을 시작하면 자신에게 할당하세요.

이슈를 추가할 때는 단일 모듈식 버그/개선/기능에 집중하려고 노력하세요.
두 이슈가 관련되거나 차단되는 경우, 결합하지 말고 링크하세요.

이슈를 가능한 한 최신 상태로 유지하려고 노력하겠지만, 이 분야의 빠른 개발 속도로 인해 일부 이슈가 최신 상태를 유지하지 않을 수 있습니다.
이러한 일이 발생하면 알려주세요.

### 💭 GitHub 토론

사용자가 사용 질문을 하고, 설계 결정을 논의하며, 새로운 기능을 제안할 수 있는 [토론](https://github.com/langchain-ai/langchain/discussions) 페이지가 있습니다.

질문에 답변할 수 있다면, 그렇게 해주세요! 이는 메인테이너가 개발 및 버그 수정에 더 많은 시간을 할애할 수 있도록 합니다.

### 🙋 도움 받기

우리의 목표는 가능한 한 간단한 개발자 설정을 제공하는 것입니다. 설정하는 데 어려움을 겪는 경우, 메인테이너에게 연락하세요!
우리는 당신이 막힘 없이 진행할 수 있도록 도와드리고, 미래의 기여자들을 위해 프로세스가 원활하도록 하고 싶습니다.

비슷한 맥락에서, 우리는 코드베이스에서 특정 린팅, 포맷팅 및 문서화 기준을 시행합니다.
이 작업이 어렵거나 귀찮다면, 메인테이너에게 도움을 요청하세요. 좋은 코드가 코드베이스에 들어가는 것을 방해하고 싶지 않습니다.

# 🌟 인정

당신의 기여가 릴리스에 반영되면, Twitter에서 당신에게 크레딧을 주고 싶습니다 (물론 원하시는 경우에만)!
언급하고 싶은 Twitter 계정이 있다면 PR에 작성하거나 다른 방법으로 알려주세요.

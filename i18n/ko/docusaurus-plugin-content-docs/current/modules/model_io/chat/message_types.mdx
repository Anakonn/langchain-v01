# 메시지 유형

ChatModels는 메시지 목록을 입력으로 받아 메시지를 반환합니다. 다양한 유형의 메시지가 있습니다. 모든 메시지에는 `role`과 `content` 속성이 있습니다. `role`은 메시지를 말하는 주체를 설명합니다. LangChain에는 다양한 역할에 대한 메시지 클래스가 있습니다. `content` 속성은 메시지의 내용을 설명합니다. 이는 다음과 같은 몇 가지 유형일 수 있습니다:

- 문자열(대부분의 모델이 이 유형의 콘텐츠를 처리함)
- 딕셔너리 목록(이는 입력 유형과 입력 위치에 대한 정보를 포함하는 멀티모달 입력에 사용됨)

또한 메시지에는 `additional_kwargs` 속성이 있습니다. 이는 공급자 특정 입력 매개변수와 같은 메시지에 대한 추가 정보를 전달하는 데 사용됩니다. 가장 잘 알려진 예는 OpenAI의 `function_call`입니다.

### HumanMessage

이는 사용자의 메시지를 나타냅니다. 일반적으로 내용만으로 구성됩니다.

### AIMessage

이는 모델의 메시지를 나타냅니다. 이에는 OpenAI 도구 호출과 같은 `additional_kwargs`가 포함될 수 있습니다.

### SystemMessage

이는 시스템 메시지를 나타내며, 모델에게 행동 방식을 알려줍니다. 일반적으로 내용만으로 구성됩니다. 모든 모델이 이를 지원하는 것은 아닙니다.

### FunctionMessage

이는 함수 호출의 결과를 나타냅니다. `role`과 `content`에 추가하여 이 메시지에는 이 결과를 생성한 함수의 이름을 나타내는 `name` 매개변수가 있습니다.

### ToolMessage

이는 도구 호출의 결과를 나타냅니다. 이는 OpenAI의 `function`과 `tool` 메시지 유형을 일치시키기 위해 FunctionMessage와 구별됩니다. `role`과 `content`에 추가하여 이 메시지에는 이 결과를 생성한 도구 호출의 ID를 나타내는 `tool_call_id` 매개변수가 있습니다.

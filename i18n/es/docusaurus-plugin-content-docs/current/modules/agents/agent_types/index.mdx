# Tipos de agentes

Esto categoriza todos los agentes disponibles a lo largo de algunas dimensiones.

**Tipo de modelo previsto**

Si este agente está diseñado para modelos de chat (recibe mensajes, produce mensajes) o LLM (recibe cadena, produce cadena). Lo principal que esto afecta es la estrategia de prompting utilizada. Puedes usar un agente con un tipo de modelo diferente al que está diseñado, pero probablemente no produzca resultados de la misma calidad.

**Admite historial de chat**

Si estos tipos de agentes admiten o no el historial de chat. Si lo hace, significa que se puede usar como un chatbot. Si no lo hace, entonces significa que es más adecuado para tareas individuales. Admitir el historial de chat generalmente requiere modelos mejores, por lo que los primeros tipos de agentes dirigidos a modelos peores pueden no admitirlo.

**Admite herramientas de entrada múltiple**

Si estos tipos de agentes admiten o no herramientas con múltiples entradas. Si una herramienta solo requiere una sola entrada, es generalmente más fácil para un LLM saber cómo invocarla. Por lo tanto, varios de los primeros tipos de agentes dirigidos a modelos peores pueden no admitirlos.

**Admite llamadas de funciones en paralelo**

Que un LLM llame a varias herramientas al mismo tiempo puede acelerar mucho a los agentes cuando hay tareas que se benefician de hacerlo. Sin embargo, es mucho más desafiante para los LLM hacer esto, por lo que algunos tipos de agentes no lo admiten.

**Parámetros de modelo requeridos**

Si este agente requiere que el modelo admita parámetros adicionales. Algunos tipos de agentes aprovechan cosas como la llamada de funciones de OpenAI, que requieren otros parámetros del modelo. Si no se requiere ninguno, significa que todo se hace a través de prompting.

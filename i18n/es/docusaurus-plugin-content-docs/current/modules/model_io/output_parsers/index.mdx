# Analizadores de salida

Los analizadores de salida son responsables de tomar la salida de un LLM y transformarla a un formato más adecuado. Esto es muy útil cuando se utilizan LLM para generar cualquier forma de datos estructurados.

Además de tener una gran colección de diferentes tipos de analizadores de salida, un beneficio distintivo de los analizadores de salida de LangChain es que muchos de ellos admiten el streaming.

## [Inicio rápido](./quick_start)

Consulte [esta guía de inicio rápido](./quick_start) para obtener una introducción a los analizadores de salida y cómo trabajar con ellos.

## Tipos de analizadores de salida

LangChain tiene muchos tipos diferentes de `analizadores de salida`.

Columnas de la tabla:

- **Nombre**: El nombre del analizador de salida
- **Admite transmisión**: Si el analizador de salida admite transmisión.
- **Tiene instrucciones de formato**: Si el analizador de salida tiene instrucciones de formato. Esto generalmente está disponible, excepto cuando (a) el esquema deseado no se especifica en el mensaje, sino en otros parámetros (como la llamada a funciones de OpenAI), o (b) cuando el OutputParser envuelve a otro OutputParser.
- **Llama a LLM**: Si este analizador de salida llama a un LLM. Esto generalmente solo lo hacen los analizadores de salida que intentan corregir la salida con formato incorrecto.
- **Tipo de entrada**: Tipo de entrada esperado. La mayoría de los analizadores de salida funcionan tanto con cadenas como con mensajes, pero algunos (como OpenAI Functions) necesitan un mensaje con kwargs específicos.
- **Tipo de salida**: El tipo de salida del objeto devuelto por el analizador.
- **Descripción**: Nuestros comentarios sobre este analizador de salida y cuándo usarlo.

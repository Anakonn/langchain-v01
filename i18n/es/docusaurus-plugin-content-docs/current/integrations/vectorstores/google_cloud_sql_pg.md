### Grupo de conexiones del motor PostgreSQL

Uno de los requisitos y argumentos para establecer Cloud SQL como un almac√©n de vectores es un objeto `PostgresEngine`. El `PostgresEngine` configura un grupo de conexiones a su base de datos de Cloud SQL, lo que permite conexiones exitosas desde su aplicaci√≥n y sigue las mejores pr√°cticas de la industria.

Para crear un `PostgresEngine` usando `PostgresEngine.from_instance()`, solo necesita proporcionar 4 cosas:

1.   `project_id` : ID del proyecto de Google Cloud donde se encuentra la instancia de Cloud SQL.
1. `region` : Regi√≥n donde se encuentra la instancia de Cloud SQL.
1. `instance` : El nombre de la instancia de Cloud SQL.
1. `database` : El nombre de la base de datos a la que conectarse en la instancia de Cloud SQL.

De forma predeterminada, se utilizar√° [autenticaci√≥n de base de datos de IAM](https://cloud.google.com/sql/docs/postgres/iam-authentication#iam-db-auth) como el m√©todo de autenticaci√≥n de base de datos. Esta biblioteca usa el principal de IAM perteneciente a las [Credenciales predeterminadas de la aplicaci√≥n (ADC)](https://cloud.google.com/docs/authentication/application-default-credentials) obtenidas del entorno.

Para obtener m√°s informaci√≥n sobre la autenticaci√≥n de base de datos de IAM, consulte:

* [Configurar una instancia para la autenticaci√≥n de base de datos de IAM](https://cloud.google.com/sql/docs/postgres/create-edit-iam-instances)
* [Administrar usuarios con autenticaci√≥n de base de datos de IAM](https://cloud.google.com/sql/docs/postgres/add-manage-iam-users)

Opcionalmente, tambi√©n se puede usar [autenticaci√≥n de base de datos integrada](https://cloud.google.com/sql/docs/postgres/built-in-authentication) mediante un nombre de usuario y una contrase√±a para acceder a la base de datos de Cloud SQL. Simplemente proporcione los argumentos opcionales `user` y `password` a `PostgresEngine.from_instance()`:

* `user` : Usuario de base de datos a usar para la autenticaci√≥n y el inicio de sesi√≥n de la base de datos integrada
* `password` : Contrase√±a de base de datos a usar para la autenticaci√≥n y el inicio de sesi√≥n de la base de datos integrada.

"**Nota**: Este tutorial demuestra la interfaz asincr√≥nica. Todos los m√©todos asincr√≥nicos tienen m√©todos sincr√≥nicos correspondientes."

### Inicializar una tabla

## Crear un Vector Store personalizado

Un Vector Store puede aprovechar los datos relacionales para filtrar las b√∫squedas de similitud.

Cree una tabla con columnas de metadatos personalizadas.

```python
from langchain_google_cloud_sql_pg import Column

# Set table name
TABLE_NAME = "vectorstore_custom"

await engine.ainit_vectorstore_table(
    table_name=TABLE_NAME,
    vector_size=768,  # VertexAI model: textembedding-gecko@latest
    metadata_columns=[Column("len", "INTEGER")],
)


# Initialize PostgresVectorStore
custom_store = await PostgresVectorStore.create(
    engine=engine,
    table_name=TABLE_NAME,
    embedding_service=embedding,
    metadata_columns=["len"],
    # Connect to a existing VectorStore by customizing the table schema:
    # id_column="uuid",
    # content_column="documents",
    # embedding_column="vectors",
)
```

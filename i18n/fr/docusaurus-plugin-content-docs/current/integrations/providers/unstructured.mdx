# Non structuré

>Le package `unstructured` de
[Unstructured.IO](https://www.unstructured.io/) extrait du texte propre à partir de documents sources bruts comme
les PDF et les documents Word.
Cette page couvre comment utiliser l'écosystème [`unstructured`](https://github.com/Unstructured-IO/unstructured)
dans LangChain.

## Installation et configuration

Si vous utilisez un chargeur qui s'exécute localement, suivez ces étapes pour faire fonctionner `unstructured` et
ses dépendances localement.

- Installez le SDK Python avec `pip install unstructured`.
    - Vous pouvez installer des dépendances spécifiques aux documents avec des extras, c'est-à-dire `pip install "unstructured[docx]"`.
    - Pour installer les dépendances pour tous les types de documents, utilisez `pip install "unstructured[all-docs]"`.
- Installez les dépendances système suivantes si elles ne sont pas déjà disponibles sur votre système.
  Selon les types de documents que vous analysez, vous n'aurez peut-être pas besoin de tous ces éléments.
    - `libmagic-dev` (détection du type de fichier)
    - `poppler-utils` (images et PDF)
    - `tesseract-ocr` (images et PDF)
    - `libreoffice` (documents MS Office)
    - `pandoc` (EPUB)

Si vous voulez être opérationnel avec moins de configuration, vous pouvez
simplement exécuter `pip install unstructured` et utiliser `UnstructuredAPIFileLoader` ou
`UnstructuredAPIFileIOLoader`. Cela traitera votre document à l'aide de l'API hébergée Unstructured.

L'`API Unstructured` nécessite des clés API pour effectuer des requêtes.
Vous pouvez demander une clé API [ici](https://unstructured.io/api-key-hosted) et commencer à l'utiliser dès aujourd'hui !
Consultez le README [ici](https://github.com/Unstructured-IO/unstructured-api) pour commencer à faire des appels API.
Nous serions ravis d'avoir vos commentaires, faites-nous savoir comment cela se passe dans notre [slack communautaire](https://join.slack.com/t/unstructuredw-kbe4326/shared_invite/zt-1x7cgo0pg-PTptXWylzPQF9xZolzCnwQ).
Et restez à l'écoute pour les améliorations de la qualité et des performances !
Consultez les instructions
[ici](https://github.com/Unstructured-IO/unstructured-api#dizzy-instructions-for-using-the-docker-image) si vous souhaitez héberger vous-même l'API Unstructured ou l'exécuter localement.

# MyScale

Cette page couvre comment utiliser la base de données vectorielle MyScale dans LangChain.
Elle est divisée en deux parties : l'installation et la configuration, puis les références aux wrappers MyScale spécifiques.

Avec MyScale, vous pouvez gérer à la fois les données structurées et non structurées (vectorisées), et effectuer des requêtes et des analyses conjointes sur les deux types de données à l'aide de SQL. De plus, l'architecture OLAP cloud-native de MyScale, construite sur ClickHouse, permet un traitement des données ultra-rapide, même sur des jeux de données massifs.

## Installation et configuration

- Installez le SDK Python avec `pip install clickhouse-connect`

### Configuration des environnements

Il existe deux façons de configurer les paramètres pour l'index MyScale.

1. Variables d'environnement

    Avant d'exécuter l'application, veuillez définir la variable d'environnement avec `export` :
    `export MYSCALE_HOST='<your-endpoints-url>' MYSCALE_PORT=<your-endpoints-port> MYSCALE_USERNAME=<your-username> MYSCALE_PASSWORD=<your-password> ...`

    Vous pouvez facilement trouver votre compte, votre mot de passe et d'autres informations sur notre SaaS. Pour plus de détails, veuillez vous référer [à ce document](https://docs.myscale.com/en/cluster-management/)
    Chaque attribut sous `MyScaleSettings` peut être défini avec le préfixe `MYSCALE_` et n'est pas sensible à la casse.

2. Créer un objet `MyScaleSettings` avec les paramètres

    ```python
    from langchain_community.vectorstores import MyScale, MyScaleSettings
    config = MyScaleSetting(host="<your-backend-url>", port=8443, ...)
    index = MyScale(embedding_function, config)
    index.add_documents(...)
    ```

que ce soit pour la recherche sémantique ou la récupération d'exemples similaires.

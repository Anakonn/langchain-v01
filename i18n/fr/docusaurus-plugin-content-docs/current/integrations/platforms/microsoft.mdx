### Azure OpenAI

>[Microsoft Azure](https://en.wikipedia.org/wiki/Microsoft_Azure), souvent appelé `Azure`, est une plateforme de cloud computing gérée par `Microsoft`, qui offre l'accès, la gestion et le développement d'applications et de services via des centres de données mondiaux. ### Azure Blob Storage

>[Azure Blob Storage](https://learn.microsoft.com/en-us/azure/storage/blobs/storage-blobs-introduction) est la solution de stockage d'objets cloud de Microsoft. Blob Storage est optimisé pour stocker de grandes quantités de données non structurées. Les données non structurées sont des données qui ne respectent pas un modèle de données ou une définition particulière, comme du texte ou des données binaires.

>[Azure Files](https://learn.microsoft.com/en-us/azure/storage/files/storage-files-introduction) offre des partages de fichiers entièrement gérés dans le cloud, accessibles via le protocole Server Message Block (`SMB`) standard de l'industrie,
> le protocole Network File System (`NFS`) et l'`API REST Azure Files`. `Azure Files` est basé sur le `Azure Blob Storage`.

`Azure Blob Storage` est conçu pour :
- Servir des images ou des documents directement à un navigateur.
- Stocker des fichiers pour un accès distribué.
- Diffuser en continu de la vidéo et de l'audio.
- Écrire dans des fichiers journaux.
- Stocker des données pour la sauvegarde et la restauration, la récupération d'urgence et l'archivage.
- Stocker des données pour une analyse par un service hébergé sur site ou dans Azure.

```bash
pip install azure-storage-blob
```

## Récupérateurs

### Azure AI Search

>[Azure AI Search](https://learn.microsoft.com/en-us/azure/search/search-what-is-azure-search) (anciennement connu sous le nom de `Azure Search` ou `Azure Cognitive Search`) est un service de recherche cloud qui fournit aux développeurs l'infrastructure, les API et les outils nécessaires pour construire une expérience de recherche riche sur du contenu privé et hétérogène dans les applications web, mobiles et d'entreprise.

>La recherche est fondamentale pour toute application qui présente du texte aux utilisateurs, où les scénarios courants incluent la recherche de catalogues ou de documents, les applications de commerce de détail en ligne ou l'exploration de données sur un contenu propriétaire. Lorsque vous créez un service de recherche, vous travaillerez avec les capacités suivantes :
>- Un moteur de recherche pour la recherche en texte intégral sur un index de recherche contenant le contenu appartenant à l'utilisateur
>- Un indexage riche, avec une analyse lexicale et un enrichissement IA en option pour l'extraction et la transformation de contenu
>- Une syntaxe de requête riche pour la recherche en texte intégral, la recherche floue, la saisie semi-automatique, la recherche géographique et plus encore
>- Programmabilité via des API REST et des bibliothèques clientes dans les kits de développement logiciel Azure
>- Intégration Azure au niveau de la couche de données, de la couche d'apprentissage automatique et des services IA

Voir [instructions de configuration](https://learn.microsoft.com/en-us/azure/search/search-create-service-portal).

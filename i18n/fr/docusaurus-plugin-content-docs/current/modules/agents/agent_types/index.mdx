# Types d'agents

Cela catégorise tous les agents disponibles selon quelques dimensions.

**Type de modèle prévu**

Si cet agent est prévu pour les modèles de chat (prend des messages en entrée, produit un message en sortie) ou les LLM (prend une chaîne de caractères en entrée, produit une chaîne de caractères en sortie). La principale chose que cela affecte est la stratégie de prompt utilisée. Vous pouvez utiliser un agent avec un type de modèle différent de celui pour lequel il est prévu, mais il est peu probable que les résultats soient de la même qualité.

**Prend en charge l'historique de la conversation**

Si ces types d'agents prennent en charge l'historique de la conversation ou non. S'ils le font, cela signifie qu'ils peuvent être utilisés comme chatbots. S'ils ne le font pas, alors cela signifie qu'ils sont plus adaptés aux tâches uniques. Prendre en charge l'historique de la conversation nécessite généralement de meilleurs modèles, donc les premiers types d'agents destinés à des modèles plus faibles peuvent ne pas le prendre en charge.

**Prend en charge les outils à entrées multiples**

Si ces types d'agents prennent en charge ou non les outils avec plusieurs entrées. Si un outil ne nécessite qu'une seule entrée, il est généralement plus facile pour un LLM de savoir comment l'invoquer. Par conséquent, plusieurs types d'agents plus anciens destinés à des modèles plus faibles peuvent ne pas les prendre en charge.

**Prend en charge l'appel de fonctions en parallèle**

Le fait qu'un LLM puisse appeler plusieurs outils en même temps peut grandement accélérer les agents, que ce soit pour des tâches qui bénéficient de cette fonctionnalité. Cependant, c'est beaucoup plus difficile pour les LLM de le faire, donc certains types d'agents ne le prennent pas en charge.

**Paramètres de modèle requis**

Si cet agent nécessite que le modèle prenne en charge des paramètres supplémentaires. Certains types d'agents tirent parti de choses comme l'appel de fonction OpenAI, qui nécessitent d'autres paramètres de modèle. S'il n'y en a pas de requis, cela signifie que tout se fait via le prompt.

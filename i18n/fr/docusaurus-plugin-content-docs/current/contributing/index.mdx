# Bienvenue aux contributeurs

Salut ! Merci d'être intéressé par la contribution à LangChain.
En tant que projet open-source dans un domaine en pleine évolution, nous sommes extrêmement ouverts aux contributions, qu'il s'agisse de nouvelles fonctionnalités, d'une infrastructure améliorée, d'une meilleure documentation ou de corrections de bugs.

## 🗺️ Lignes directrices

### 👩‍💻 Façons de contribuer

Il existe de nombreuses façons de contribuer à LangChain. Voici quelques-unes des façons les plus courantes pour les gens de contribuer :

- [**Documentation**](/docs/contributing/documentation/style_guide) : Aidez-nous à améliorer notre documentation, y compris celle-ci !
- [**Code**](./code.mdx) : Aidez-nous à écrire du code, à corriger des bugs ou à améliorer notre infrastructure.
- [**Intégrations**](integrations.mdx) : Aidez-nous à nous intégrer à vos fournisseurs et outils préférés.
- [**Discussions**](https://github.com/langchain-ai/langchain/discussions) : Aidez à répondre aux questions d'utilisation et à discuter des problèmes avec les utilisateurs.

### 🚩 Problèmes GitHub

Notre page [problèmes](https://github.com/langchain-ai/langchain/issues) est tenue à jour avec les bugs, les améliorations et les demandes de fonctionnalités.

Il existe une taxonomie d'étiquettes pour aider au tri et à la découverte des problèmes d'intérêt. Veuillez les utiliser pour aider à organiser les problèmes.

Si vous commencez à travailler sur un problème, veuillez vous l'attribuer.

Si vous ajoutez un problème, essayez de le garder concentré sur un seul bug/amélioration/fonctionnalité modulaire.
Si deux problèmes sont liés ou bloquants, veuillez les lier plutôt que de les combiner.

Nous essaierons de garder ces problèmes aussi à jour que possible, bien qu'avec le rythme de développement rapide dans ce domaine, certains puissent devenir obsolètes.
Si vous remarquez que cela se produit, n'hésitez pas à nous le faire savoir.

### 💭 Discussions GitHub

Nous avons une page [discussions](https://github.com/langchain-ai/langchain/discussions) où les utilisateurs peuvent poser des questions d'utilisation, discuter des décisions de conception et proposer de nouvelles fonctionnalités.

Si vous êtes en mesure d'aider à répondre aux questions, n'hésitez pas à le faire ! Cela permettra aux mainteneurs de se concentrer davantage sur le développement et la correction des bugs.

### 🙋 Obtenir de l'aide

Notre objectif est d'avoir le processus de configuration du développeur le plus simple possible. Si vous rencontrez des difficultés pour vous configurer, n'hésitez pas à contacter un mainteneur ! Nous ne voulons pas seulement vous débloquer, mais nous voulons également nous assurer que le processus est fluide pour les futurs contributeurs.

Dans le même ordre d'idées, nous appliquons certaines normes de linting, de formatage et de documentation dans la base de code.
Si vous les trouvez difficiles (ou même juste agaçantes) à utiliser, n'hésitez pas à contacter un mainteneur pour obtenir de l'aide - nous ne voulons pas que cela entrave l'intégration de bon code dans la base de code.

# 🌟 Reconnaissance

Si votre contribution a été intégrée dans une version, nous voudrons vous en donner crédit sur Twitter (seulement si vous le voulez cependant) !
Si vous avez un compte Twitter que vous aimeriez que nous mentionnions, n'hésitez pas à nous le faire savoir dans la demande de tirage ou par un autre moyen.

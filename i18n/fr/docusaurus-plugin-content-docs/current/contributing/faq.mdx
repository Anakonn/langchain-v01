# Questions fréquemment posées

## Pull Requests (PR)

### Comment puis-je autoriser les responsables à modifier ma PR ?

Lorsque vous soumettez une pull request, des modifications supplémentaires peuvent être nécessaires avant de la fusionner. Souvent, il est plus efficace pour les responsables d'effectuer ces changements eux-mêmes avant la fusion, plutôt que de vous le demander lors de l'examen du code.

Par défaut, la plupart des pull requests auront un badge `✅ Les responsables sont autorisés à modifier cette pull request.` dans la barre latérale de droite.

Si vous ne voyez pas ce badge, il se peut que ce paramètre soit désactivé pour le fork à partir duquel vous effectuez la pull request. Consultez [cette page de la documentation Github](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/allowing-changes-to-a-pull-request-branch-created-from-a-fork) pour plus d'informations.

Notamment, Github ne permet pas d'activer ce paramètre pour les forks dans les **organisations** ([problème](https://github.com/orgs/community/discussions/5634)). Si vous travaillez dans une organisation, nous vous recommandons de soumettre votre PR à partir d'un fork personnel afin d'activer ce paramètre.

# Structure du dépôt

Si vous prévoyez de contribuer au code ou à la documentation de LangChain, il peut être utile de comprendre la structure de haut niveau du dépôt.

LangChain est organisé comme un [monorep](https://en.wikipedia.org/wiki/Monorepo) qui contient plusieurs packages.

Voici la structure visualisée sous forme d'arborescence :

```text
.
├── cookbook # Tutorials and examples
├── docs # Contains content for the documentation here: https://python.langchain.com/
├── libs
│   ├── langchain # Main package
│   │   ├── tests/unit_tests # Unit tests (present in each package not shown for brevity)
│   │   ├── tests/integration_tests # Integration tests (present in each package not shown for brevity)
│   ├── langchain-community # Third-party integrations
│   ├── langchain-core # Base interfaces for key abstractions
│   ├── langchain-experimental # Experimental components and chains
│   ├── partners
│       ├── langchain-partner-1
│       ├── langchain-partner-2
│       ├── ...
│
├── templates # A collection of easily deployable reference architectures for a wide variety of tasks.
```

Le répertoire racine contient également les fichiers suivants :

* `pyproject.toml` : Dépendances pour la construction de la documentation et le linting des documents, du cookbook.
* `Makefile` : Un fichier contenant des raccourcis pour la construction, le linting et la documentation et le cookbook.

Il y a d'autres fichiers au niveau du répertoire racine, mais leur présence devrait être évidente. N'hésitez pas à les parcourir !

## Documentation

Le répertoire `/docs` contient le contenu de la documentation qui est affiché sur https://python.langchain.com/ et la référence de l'API associée https://api.python.langchain.com/en/latest/langchain_api_reference.html.

Consultez les [directives de documentation](/docs/contributing/documentation/style_guide) pour apprendre comment contribuer à la documentation.
